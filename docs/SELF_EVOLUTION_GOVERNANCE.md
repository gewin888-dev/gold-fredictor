# 自我进化功能治理说明

本系统的自我进化目标是提高黄金多空评分和金价预测参考质量，不生成交易建议。正式交付默认只允许自动生成候选版本，模型激活需要人工审核。

## 当前闭环

1. 数据采集任务刷新宏观、金价、持仓、情绪、外部市场结构等数据。
2. 系统使用当前激活评分参数计算并保存 `GoldScoreSnapshot`。
3. 预测模块基于可信评分源和历史金价生成 `GoldPredictionSnapshot`。
4. 到期预测由 `evaluate_due_predictions()` 写入 `GoldPredictionEvaluation`。
5. `auto_optimize_job()` 根据数据库中的自动优化开关决定是否生成候选评分参数或候选预测模型。
6. 候选版本进入版本表，默认不激活。
7. 仪表盘“模型健康”展示当前版本、样本状态、候选状态和不能进化的原因。
8. 人工通过 API 或仪表盘激活候选，也可以恢复默认评分规则。

## 安全边界

- 自动优化开关写入 `app_settings` 表，不改写 `.env`。
- `.env` 只保存密钥和部署默认值。
- `AUTO_ACTIVATE_OPTIMIZED_PARAMS` 与 `AUTO_ACTIVATE_PREDICTION_MODEL` 默认关闭。
- `auto_evolve_if_needed()` 只保存候选，不自动激活。
- 外部数据缺失时返回结构化原因，不阻塞 API 启动、调度器和评分流程。

## 评分候选门控

评分参数优化不再只看方向命中率，还会记录：

- 样本数
- 信号覆盖率
- 多头信号数
- 空头信号数
- 相对 baseline 提升
- 平均方向信号收益
- 最差分位收益
- 最近窗口稳定性

默认激活建议门槛：

- 样本数不少于 120
- 信号覆盖率不少于 20%
- 命中率相对 baseline 至少提升 3 个百分点
- 多头和空头样本都不能过少
- 最近窗口不能明显退化

即便候选满足门槛，正式交付仍建议人工审核后激活。

## 预测模型门控

预测模型优化使用 walk-forward 多 horizon 评估。候选模型会记录：

- 优化综合分
- 加权 MAPE
- 加权 MAE
- 加权方向准确率
- 有效 horizon 数
- 样本数

当前数据库预测评估样本不足时，页面会显示“不足以进化”，不会误报模型已经完成闭环。

## 数据不足时的降级

以下因子可以展示但默认不参与评分，直到接入稳定来源或人工录入：

- COMEX 注册库存
- COMEX 期限结构
- 地缘风险强度
- 中印实物需求

这些因子在目录和页面中保留灰色状态，并给出不能评分的原因。ETF 资金流、期权隐含波动率等指标在可信来源入库后可参与评分和优化。

## 真实使用场景

日常运行时，建议流程如下：

1. 保持自动采集和评分任务运行。
2. 定期保存预测快照，等待不同 horizon 到期。
3. 先积累足够预测评估样本，再观察预测模型候选。
4. 每周或每月生成评分候选参数。
5. 在仪表盘查看候选相对 baseline 的改善和样本结构。
6. 只在候选解释充分且没有明显过拟合时人工激活。
7. 激活后继续观察健康面板和后续回测表现。
8. 若表现退化，使用 `/score/params/deactivate` 恢复默认评分。

## 后续改进建议

- 增加模型版本审计表，记录激活人、激活原因、候选指标和回滚原因。
- 将评分因子进一步抽象为显式 registry，使实时评分、历史回测和 UI 因子卡片完全共用同一份定义。
- 增加 regime 分段评估，区分高通胀、降息周期、美元流动性收缩、地缘冲突等阶段。
- 引入人工录入页面，维护 COMEX 期限结构、地缘风险和实物需求数据。
- 对自动优化候选增加过拟合检测，例如训练/验证窗口分离和 rolling stability 报告。
- 增加飞书推送中的模型健康摘要，只提示风险和状态，不给出买卖建议。
