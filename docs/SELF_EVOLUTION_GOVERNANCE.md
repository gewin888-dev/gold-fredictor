# 自我进化功能治理说明

本系统用于验证“零代码基础借助 AI 开发黄金预测系统”的可行性，目标是持续对标真实黄金走势。系统不用于黄金买卖参考。

默认策略是无人值守：自动生成候选、自动评估、自动修复、自动回滚，并在候选通过严格质量门控后自动激活。页面和 API 保留手动操作入口，只用于调试、复盘和紧急覆盖，不作为日常运行的必需步骤。

## 当前闭环

1. 数据采集任务刷新宏观、金价、持仓、情绪、外部市场结构等数据。
2. 系统使用当前激活评分参数计算并保存 `GoldScoreSnapshot`。
3. 预测模块基于可信评分源和历史金价生成 `GoldPredictionSnapshot`。
4. 到期预测由 `evaluate_due_predictions()` 写入 `GoldPredictionEvaluation`。
5. `run_self_healing_cycle()` 自动补评估、检查退化、必要时回滚预测模型。
6. `auto_evolve_if_needed()` 和 `auto_optimize_job()` 自动搜索评分参数和预测模型候选。
7. 候选版本进入版本表；若样本量、命中率、方向准确率、MAPE、baseline 提升和近期退化检查通过，则自动激活。
8. 仪表盘展示当前版本、样本状态、候选状态、不能进化的原因和最近激活/回滚记录。

## 质量门控

- `AUTO_EVOLUTION_FULL_AUTO=true` 表示无人值守，不表示放宽质量标准。
- `AUTO_ACTIVATE_OPTIMIZED_PARAMS=true` 与 `AUTO_ACTIVATE_PREDICTION_MODEL=true` 允许达标候选自动激活。
- 激活前必须满足配置门槛，例如 `AUTO_OPTIMIZE_MIN_HIT_RATE`、`AUTO_PREDICTION_MIN_SAMPLES`、`AUTO_PREDICTION_MIN_SCORE`、`AUTO_PREDICTION_MAX_MAPE_PCT`、`AUTO_PREDICTION_MIN_DIRECTION_ACCURACY`。
- 外部数据缺失时返回结构化原因，不阻塞 API 启动、调度器和评分流程。

## 评分候选门控

评分参数优化不只看方向命中率，还会记录：

- 样本数
- 信号覆盖率
- 多头信号数
- 空头信号数
- 相对 baseline 提升
- 平均方向信号收益
- 最差分位收益
- 最近窗口稳定性

默认激活门槛：

- 样本数不少于 120
- 信号覆盖率不少于 20%
- 命中率相对 baseline 至少提升 3 个百分点
- 多头和空头样本都不能过少
- 最近窗口不能明显退化

候选满足门槛后可自动激活；系统会记录版本、指标和激活原因，便于复盘。

## 预测模型门控

预测模型优化使用 walk-forward 多 horizon 评估。候选模型会记录：

- 优化综合分
- 加权 MAPE
- 加权 MAE
- 加权方向准确率
- 有效 horizon 数
- 样本数

当前数据库预测评估样本不足时，页面会显示“不足以进化”，不会误报模型已经完成闭环，也不会为了全自动而降低准确性要求。

## 数据不足时的降级

以下因子可以展示但默认不参与评分，直到接入稳定来源或手动/授权录入：

- COMEX 注册库存
- COMEX 期限结构
- 地缘风险强度
- 中印实物需求

这些因子在目录和页面中保留灰色状态，并给出不能评分的原因。ETF 资金流、期权隐含波动率等指标在可信来源入库后可参与评分和优化。

## 运行原则

1. 保持自动采集和评分任务运行。
2. 持续保存预测快照，等待不同 horizon 到期。
3. 自动补评估、自动生成候选、自动门控激活。
4. 激活后继续观察健康面板和后续回测表现。
5. 若表现退化，系统自动回滚；手动回滚入口只作为调试和紧急覆盖。

## 后续改进建议

- 增加 regime 分段评估，区分高通胀、降息周期、美元流动性收缩、地缘冲突等阶段。
- 将评分因子进一步抽象为显式 registry，使实时评分、历史回测和 UI 因子卡片完全共用同一份定义。
- 对自动优化候选增加训练/验证窗口分离和 rolling stability 报告。
- 增加飞书推送中的模型健康摘要，只提示系统状态和预测能力验证结果。
